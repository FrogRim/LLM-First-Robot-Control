## 📊 현재 학습 결과 재확인

### 손실 곡선
```
Epoch 1/4: loss=1.234 → 0.876 (eval_loss: 0.912)
Epoch 2/4: loss=0.723 → 0.542 (eval_loss: 0.589)

```


## 🎯 왜 Epoch 4로 과적합이 없는가?

### 1. LoRA의 특성 (가장 중요한 이유)

**LoRA (Low-Rank Adaptation)**:
- 전체 파라미터 중 **0.14%만 학습** (약 20만 개 파라미터)
- 원본 모델의 지식은 고정, **작은 어댑터만** 학습
- 본질적으로 **Regularization 효과**가 강함

**증거**:
```python
# Qwen2.5-14B 전체 파라미터: 14.7B (147억 개)
# LoRA 학습 파라미터: r=16, 2M개 (0.14%)
# → 과적합 위험이 매우 낮음
```

### 2. 데이터 규모가 충분

**현재 데이터셋**:
- Train: 446개 샘플 (v3)
- LoRA에서는 **데이터/파라미터 비율**이 중요하지 않음
- 전통적 ML의 "10-100배" 규칙이 적용되지 않음

**왜?**
- LoRA는 **지식 증류**처럼 작동
- 기존 지식은 유지하고 **도메인 특화**만 학습
- 446개 샘플로 충분한 도메인 특화 가능

### 3. 학습 곡선의 품질

**양호한 수렴**:
- ✅ Train/Eval loss 함께 감소 (과적합 패턴 아님)
- ✅ 안정적인 감소 (불규칙한 oscillation 없음)
- ✅ 최종 loss 안정화 (0.278 → 더 이상 감소 없을 듯)

**만약 과적합이었다면**:
- ❌ Train loss만 계속 감소
- ❌ Eval loss 증가 또는 plateau
- ❌ Train/Eval gap 커짐

### 4. 도메인 특화 학습의 특성

**물리 도메인 학습**:
- 기존 언어 모델링 능력 유지
- **물리 법칙, 재료 속성**만 추가 학습
- 과적합보다는 **언더피팅**이 더 쉬움

**증거**: 일반화 테스트 결과
```
재료명 없는 추론: 71.4% (충분히 좋은 성능)
→ 과적합이라면 train만 좋고 일반화 안 됨
```
