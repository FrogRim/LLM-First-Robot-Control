## 1. 연구 개요 (Introduction / Background)

### 연구 주제
본 연구는 대형 언어 모델(LLM)을 활용하여 자연어 명령으로부터 물리 속성을 추론하고 로봇 제어 파라미터를 자동 생성하는 LLM-First 아키텍처 기반 로봇 제어 시스템을 개발하는 것을 목표로 한다.

### 연구 동기
기존 로봇 제어 시스템은 주로 센서 기반 물리 속성 추정과 하드코딩된 제어 규칙에 의존하여, 자연어 명령에 대한 유연한 대응이 부족한 한계를 가지고 있다. 본 연구는 이러한 한계를 극복하기 위해 LLM의 강력한 추론 능력을 활용하여, 텍스트 명령으로부터 물리 속성을 자동 추론하고 적절한 제어 파라미터를 생성하는 혁신적인 접근법을 제시한다.

### 핵심 목표
1. **LLM-First 아키텍처 구현**: 자연어 명령을 직접 입력으로 받아 물리 속성 추론 및 제어 파라미터 생성
2. **데이터 변환 파이프라인 구축**: DROID 공개 데이터셋을 Genesis AI 시뮬레이션 환경으로 변환
3. **고품질 학습 데이터 생성**: 물리 도메인 특화 데이터 증강 및 품질 관리
4. **실시간 성능 검증**: JSON 파싱률 100% 및 최적화된 추론 시간 달성

## 2. 관련 연구 정리 (Related Work)

### LLM-First 접근
최근 LLM-First 아키텍처(LangChain, LangGraph, AutoGen 등)는 복잡한 작업을 자연어로 분해하고 단계별로 해결하는 패러다임을 제시한다. 기존 연구들은 주로 일반적인 작업 자동화에 초점을 맞추었으나, 본 연구는 이를 로봇 제어 분야로 확장하여 물리 속성 추론에 적용한다.

### 물리 시뮬레이션과의 연계 연구
Genesis AI, NVIDIA Isaac Sim, Unreal Chaos 등 물리 시뮬레이션 엔진들은 고정밀 물리 시뮬레이션을 제공하지만, 자연어 인터페이스와의 통합이 부족하다. 본 연구는 LLM을 통해 이러한 시뮬레이션 환경을 자연어로 제어할 수 있는 다리를 구축한다.

### 기존 물리 속성 추정 방법
기존 방법들은 주로 비전 센서(RGB-D 카메라, LiDAR)와 머신러닝 기반 물리 속성 추정에 의존한다. 반면 본 연구는 언어 기반 접근을 통해 센서 의존도를 낮추고, 추론 과정의 설명 가능성을 높인다.

### 본 연구의 차별점
- **데이터 기반 접근**: 공개 데이터셋(DROID)을 활용한 대규모 학습 데이터 생성
- **End-to-End 통합**: 자연어 명령 → 물리 추론 → 제어 파라미터 → 시뮬레이션 검증
- **실시간성 확보**: JSON 파싱률 100%로 실용적인 로봇 제어 시스템 구현

## 3. 연구 방법 (Methodology)

### 전체 시스템 구조도

```
자연어 명령 → LLM 추론 → 물리 파라미터 JSON → Genesis AI 시뮬레이션
     ↓             ↓              ↓                      ↓
  "Pick up..."  Qwen2.5-14B   {"grip_force": 0.2}   3D 시각화 및 검증
```

- **Sub-agent 구조**: 단일 LLM 에이전트가 물리 분석, 제어 파라미터 생성, 추론 설명을 통합 수행
- **데이터 흐름**: DROID 데이터셋 → 변환 파이프라인 → 학습 데이터 → 미세 조정 모델 → 추론 → 시뮬레이션 검증

### LLM 설정
- **모델**: Qwen2.5-14B-Instruct (최신 14B 파라미터 모델)
- **프롬프트 전략**: 강화된 시스템 프롬프트로 JSON 출력 강제 및 스키마 정의
- **LoRA/QLoRA 적용**: QLoRA (4-bit 양자화)로 메모리 효율적 미세 조정
- **파인튜닝 데이터셋 구조**: Alpaca 포맷 기반 350개 샘플 (물리 속성 분석 + 제어 파라미터 생성)

### 물리 데이터셋 구성
- **생성한 데이터 수**: 초기 60개 → 증강 후 350개 샘플 (5.8배 증가)
- **라벨링 기준**: 재료별 물리 프로파일 (plastic, metal, glass, wood, rubber, ceramic, fabric)
- **품질 관리 방식**: 다중 검증 (JSON 파싱 성공률, 물리 일관성, 추론 정확도)

### 시뮬레이션 환경
- **프레임워크**: Genesis AI 0.3.3 (강체 동역학 시뮬레이션)
- **로봇 플랫폼**: Franka Emika Panda (ROS2 호환)
- **실험 파라미터**:
  - 시뮬레이션 FPS: 40-60
  - 추론 지연: 평균 30초 (최적화 목표: <200ms)
  - 제어 루프: LLM 추론 → 파라미터 생성 → 시뮬레이션 실행

## 4. 실험 및 결과 (Experiment & Results)

### 테스트 시나리오 예시
1. **가벼운 플라스틱 물체**: "Pick up the plastic bottle gently" → grip_force: 0.2N, lift_speed: 0.71m/s
2. **무거운 금속 물체**: "Grab the heavy metal tool firmly" → grip_force: 1.0N, lift_speed: 0.3m/s
3. **취약한 유리 컵**: "Lift the glass cup very slowly" → grip_force: 0.2N, lift_speed: 0.2m/s

### 평가 지표
- **JSON 파싱률**: 100% (30/30 샘플 성공)
- **추론 시간**: 평균 30초 (벤치마크: 96ms)
- **재료별 정확도**: 100% (7가지 재료 모두 정확 추론)
- **평균 신뢰도**: 0.85

### 비교 대상
- **기존 센서 기반 방법**: 물리 속성 추정 정확도 85-90%
- **하드코딩 규칙 기반**: 유연성 부족, 새로운 시나리오 대응 한계
- **본 연구**: 100% 추론 성공률, 자연어 유연성 확보

### 결과 요약표

### 주요 관찰사항
- **재료별 차별화**: Metal은 높은 grip_force(0.92N), Glass는 낮은 lift_speed(0.2m/s)로 정확한 전략 적용
- **다중 제약 처리**: "gently and carefully" 같은 복합 명령을 성공적으로 해석
- **실시간성 한계**: 현재 30초 추론 시간으로 실시간 제어에 부적합 → 모델 경량화 필요
