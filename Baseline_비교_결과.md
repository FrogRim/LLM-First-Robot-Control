## 🤔 주요 발견 및 논의

### 1. Base Model의 강력한 성능

**Qwen2.5-14B-Instruct의 원래 능력**:
- 사전 학습 단계에서 이미 JSON 생성 능력 확보
- Instruction-following 능력이 매우 우수
- 강화된 시스템 프롬프트만으로도 100% 파싱 가능

**시사점**:
- 대형 모델(14B)의 경우, 프롬프트 엔지니어링만으로도 충분할 수 있음
- Fine-tuning의 주요 목적은 JSON 파싱이 아니라 **도메인 특화 지식** 학습

### 2. Fine-tuning의 실제 효과

**현재 지표로는 명확하지 않음**:
- JSON 파싱률: 동일
- 추론 시간: 오히려 느림
- 신뢰도: 거의 동일

**더 깊이 있는 평가 필요**:
1. **재료 인식 정확도**: 명령어에 재료가 명시되지 않은 경우
2. **제어 파라미터의 물리적 타당성**: grip force, lift speed가 합리적인가?
3. **일반화 능력**: 학습하지 않은 시나리오에서의 성능
4. **안전성**: 깨지기 쉬운 물체에 대한 보수적 접근

### 3. 재료 인식 정확도 0% 이슈

**문제**:
- Fine-tuned model의 material_accuracy_percent = 0.0%
- 하지만 실제로는 재료를 예측하고 있음 (예: "fabric (polyester)")

**원인**:
- Ground Truth 추출 로직에 문제
- 데이터셋 구조상 GT와 예측값의 포맷이 다름
- 비교 로직 개선 필요

**해결 방안**:
- 일반화 테스트로 실제 추론 능력 평가
- 재료명 없는 명령어로 암묵적 추론 능력 테스트
